# Learning Log: Iteration 1

### API Analysis
The Open-Meteo API returns a well-structured JSON object. I found that the data is nested within `current` and `daily` objects. The response includes a `weather_code`, which is a numerical value that I will need to map to human-readable text (e.g., 0 for "Clear Sky") in the next iteration.

### Key Fields Used
* **temperature_2m**: Used to display the primary current temperature.
* **relative_humidity_2m**: Used to provide additional weather context.
* **Coordinates**: I hard-coded the latitude (**42.36**) and longitude (**-71.06**) for **Boston**.

### Challenges & Solutions
The trickiest part was handling the **async/await** logic. I had to ensure the app didn't crash while waiting for the data to load. I solved this by implementing a simple "Loading..." state in the HTML that hides once the `fetch()` call successfully populates the UI with data.
