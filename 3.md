# Learning Log: Iteration 3

**Date:** May 1, 2026
**Goal:** Interactivity and Polish

### Interactive Features
I added a search functionality using the **Open-Meteo Geocoding API**. This allows users to search for any city globally, not just a hard-coded location. I also implemented `localStorage` so that when the page is refreshed, the app remembers the last city the user searched for.

### Error Handling & UX
I implemented a `try...catch` block to handle scenarios where a user enters a non-existent city or the API is unreachable. A friendly error message now appears instead of the app simply failing to update. I also added a `loading` state to ensure the user knows data is being fetched during the brief delay.

### Final Polish
I refined the CSS to match a professional, minimalist portfolio piece. By using a grid layout for the forecast cards and consistent iconography, the information is much easier to scan. I also ensured the timezones were handled automatically by the API (`timezone=auto`) so the day names are always correct for the searched location.
